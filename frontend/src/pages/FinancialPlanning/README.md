# Модуль: Финансовое планирование (Бюджетирование)

Модуль предназначен для планирования расходов подразделений (департаментов) организации, распределения денежных средств по статьям расходов и согласования бюджетов.

## 📋 Основной функционал

1. **Список бюджетов** — отображает финансовые планы департаментов по категориям (например, *Salaries (Зарплата)*, *Campaigns (Реклама)*, *Training (Обучение)*) и отчетным периодам (например, кварталы: `2026-Q1`).
2. **Параметры бюджета:**
   - **Запланировано (Planned):** Первоначальная планируемая сумма расходов.
   - **Лимиты (Limits):** Максимально разрешенный лимит затрат (по умолчанию равен запланированной сумме, если не указано иное).
   - **Утверждено (Approved):** Фактически согласованная руководством сумма (заполняется после подтверждения).
   - **Потрачено (Spent):** Реально израсходованные средства за период по данной статье.
   - **Остаток (Remaining):** Доступный остаток бюджета (вычисляется автоматически как разница между утвержденной или запланированной суммой и фактически потраченной).
3. **Статус согласования (Approval Status):**
   - `pending` — бюджет ожидает проверки и утверждения руководством.
   - `approved` — бюджет согласован и готов к расходу средств.

---

## 🔑 Разграничение прав (RBAC)

- **Просмотр финансовых планов:**
  - Доступен авторизованным пользователям системы. Фильтрация данных может осуществляться по должностям (в зависимости от роли пользователя).
- **Создание бюджетов:**
  - Разрешено только ролям **Admin** и **Department Head**.
  - При создании бэкенд проверяет существование указанного департамента, а также валидирует плановую сумму: она должна быть числом больше нуля.
- **Утверждение бюджета (Approve):**
  - Разрешено только ролям **Admin** и **Department Head**. Руководитель подтверждает бюджет, меняя его статус с `pending` на `approved`.
- **Удаление бюджета:**
  - Разрешено исключительно роли **Admin**.

---

## 📡 API-Эндпоинты

- `GET /api/finances` — получить список всех бюджетов (поддерживает фильтрацию по должностям `?responsiblePostId=` и периоду `?period=`).
- `POST /api/finances` — создать новый бюджет департамента (параметры запроса: `departmentId`, `responsiblePostId`, `category`, `period`, `planned`, `limits`).
- `POST /api/finances/:id/approve` — утвердить бюджет, установив статус `approved` (только Admin или Department Head).
- `DELETE /api/finances/:id` — полностью удалить статью бюджета из базы данных (только Admin).
