# Модуль: Пользователи, Авторизация и Лог Аудита

Этот модуль отвечает за безопасность системы, разграничение прав доступа на основе ролей и протоколирование всех значимых действий пользователей.

## 🔐 Авторизация

- **Регистрация:** `POST /api/auth/signup` — имя, email, пароль (≥6 символов), сразу JWT без подтверждения почты.
- Вход: `POST /api/auth/login` — email и пароль → JWT на **7 дней** (`Authorization: Bearer <token>`).
- При смене роли на бэкенде может прийти заголовок `X-Token-Refresh-Required: true` — нужно перелогиниться.

### API

- `POST /api/auth/login` — вход
- `GET /api/auth/me` — текущая сессия

---

## 👥 Ролевая модель (RBAC)

Роли вычисляются по должностям в оргструктуре:

1. **Admin** — полный доступ, управление пользователями
2. **Inspector** — только чтение/аудит
3. **Department Head** — ветка оргструктуры
4. **Section Head** — отдел
5. **Employee** — базовые права

При нескольких должностях берётся наивысшая роль.

---

## 📝 Лог аудита

`appendAuditLog()` фиксирует создание/изменение/удаление сущностей. Лента на Dashboard; в карточке должности — история по посту.

### API (управление пользователями, Admin)

- `GET /api/org/users`
- `GET /api/org/users/with-roles`
- `POST /api/org/users/:id/make-admin`
- `POST /api/org/users/:id/remove-admin`
- `DELETE /api/org/users/:id`

### Лог аудита

- `GET /api/audit/recent`
