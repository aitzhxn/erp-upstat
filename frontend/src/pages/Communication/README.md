# Модуль: Внутренняя почта (Коммуникация)

Модуль представляет собой защищенный корпоративный почтовый сервис. Его ключевая особенность заключается в том, что отправка писем осуществляется не конкретным сотрудникам по ФИО, а на **должностные ящики (посты оргструктуры)**. При смене сотрудника на должности вся история переписки по его посту автоматически сохраняется и переходит новому сотруднику.

## 📋 Основной функционал

1. **Система ящиков (Mailbox):** Письмо доставляется на конкретный `recipientPostId`.
2. **Папки писем:**
   - **Входящие (Inbox):** Письма, отправленные на должности текущего пользователя.
   - **Отправленные (Sent):** Письма, отправленные с должностей текущего пользователя.
   - **Архив (Archive):** Важные архивные письма.
3. **Отправка писем:** Форма создания письма с выбором должности получателя, темы, текста и прикреплением файлов.
4. **Вложения (Attachments):** Загрузка файлов и их скачивание другими участниками переписки.
5. **Групповые операции:** Выделение нескольких писем галочками с возможностью массового архивирования или удаления.
6. **Очистка папки:** Кнопка быстрой очистки содержимого папки для выбранной должности.
7. **Индикатор непрочитанных писем:** Отображает число непрочитанных писем во всех ящиках пользователя в реальном времени.

## 🔒 Безопасность и хранение файлов (Вложения)

Система загрузки файлов имеет встроенные многоуровневые проверки безопасности:
1. **Директория хранения:** Все файлы загружаются на бэкенд в папку `backend/uploads/` под зашифрованными уникальными именами: `att_<timestamp>_<random>.<ext>` (это предотвращает перезапись файлов с одинаковыми именами).
2. **Ограничение размера:** Лимит на размер одного файла составляет **10 МБ** (настраивается в Multer).
3. **Разрешенные типы файлов (MIME-типы):** Разрешены только безопасные форматы:
   - Документы: `application/pdf`, `.txt`, `.doc`/`.docx`, `.xls`/`.xlsx`.
   - Изображения: `.jpg`/`.jpeg`, `.png`, `.gif`, `.webp`.
   *Исполняемые файлы (`.exe`, `.sh`, `.bat`) и другие потенциально опасные типы блокируются на уровне фильтрации файлов.*
4. **Защита от выхода за пределы директории (Path Traversal Prevention):**
   При скачивании файла бэкенд вычисляет абсолютный путь файла на диске и сравнивает его с абсолютным путем папки `uploads/` (`resolvedAbsPath.startsWith(resolvedUploadsDir)`). Это предотвращает скачивание системных файлов ОС (например, `/etc/passwd`) через подмену пути в параметре запроса.

## 🔑 Доступ и права (RBAC)

- **Чтение писем и скачивание файлов:**
  - Пользователь может читать письма и скачивать их вложения **только если он занимает должность отправителя или получателя** данного письма.
  - Попытка скачать файл из чужой переписки возвращает ошибку `403 Forbidden`.
- **Отправка писем:**
  - Письмо отправляется с одной из должностей, которую пользователь занимает в данный момент (`senderPostId` проверяется на соответствие списку `getPostsForUser`).

## 📡 API-Эндпоинты

- `GET /api/communication/unread-count` — получить количество непрочитанных писем текущего пользователя.
- `GET /api/communication` — получить список писем для выбранной должности (`?postId=`) и папки (`?folder=inbox|sent|archive`). Возвращает письма вместе со списком метаданных вложений.
- `GET /api/communication/messages/:id` — получить детальное содержимое одного письма (включая вложения) с проверкой прав доступа.
- `PATCH /api/communication/messages/:id/read` — отметить письмо как прочитанное.
- `POST /api/communication/send` — отправить письмо. Поддерживает Multipart Form Data для прикрепления файлов (до 10 файлов).
- `GET /api/communication/attachments/:id` — скачать файл вложения с проверкой прав доступа и защитой от Path Traversal.
- `POST /api/communication/messages/archive` — архивировать выбранные письма (перемещает в папку `archive` списком `{ ids }`).
- `POST /api/communication/messages/delete` — удалить выбранные письма (перемещает в корзину/удаляет списком `{ ids }`).
- `POST /api/communication/clear` — очистить выбранную папку почтового ящика должности (`{ postId, folder }`).
