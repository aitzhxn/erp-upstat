# Модуль: Статистика и показатели эффективности

Модуль статистики служит для измерения эффективности работы компании. Показатели (метрики) привязаны к должностям оргсхемы, что позволяет оцифровать работу каждого поста.

## 📋 Основной функционал и терминология

В основе модуля лежат четыре главные сущности:
1. **Определение метрики (Metric Definition):** Глобальный тип показателя, например: `revenue` (Валовой доход, руб), `completedTasks` (Выполненные задачи, шт), `calls` (Исходящие звонки, шт).
2. **Назначение метрики на должность (Metric-to-Post):** Связывает должность с метрикой. Для каждой связки можно назначить:
   - **Ответственного сотрудника** (`responsibleUserId`).
   - **Ежедневную цель** (`dailyTarget`), например: делать 10 звонков в день.
3. **Квоты (Quotas):** Плановое (целевое) значение метрики на определенный период (неделя, месяц, квартал, год) для конкретной должности.
4. **Записи статистики (Statistics Records / Fact):** Фактически достигнутые показатели за период.

### Режимы работы
- **Конструктор (Constructor View):** Интерфейс для сопоставления планов (квот) и фактов за выбранный период с автоматическим подсчетом дефицита/профицита.
- **Ежедневный учет (Daily Tracking):** Удобная сетка быстрого ввода показателей за дни текущей недели. Поддерживает автосохранение по событию `onBlur` (утеря фокуса с поля ввода).
- **Сводная сетка (Unified Grid):** Таблица показателей всей организации, агрегированная по периодам.
- **Аналитическая панель (Analytics Drawer):**
   - График «План vs Факт» за последние 7 дней.
   - График динамики за последние 30 дней.
   - **WoW Growth (Week-over-Week):** Автоматический расчет темпа роста показателя по отношению к предыдущей неделе в процентах.

## 🔑 Разграничение прав (RBAC)

- **Глобальные метрики:**
  - Создавать новые типы метрик и удалять их может только **Admin**.
  - Удаление метрики блокируется базой данных, если она уже назначена должностям (ошибка `METRIC_IN_USE`).
- **Назначение показателей должностям (Metric-to-Post):**
  - Разрешено ролям **Admin** и **Department Head** (для должностей своего отделения).
- **Ввод ежедневной статистики (Daily Entry):**
  - Записать значение в ячейку могут:
    1. Пользователь с ролью **Admin**.
    2. Руководитель отделения (**Department Head**) для подчиненных должностей.
    3. Сотрудник, непосредственно назначенный **ответственным** (`responsibleUserId`) за данную метрику на этой должности.
- **Управление планами (Квотами):**
  - Создавать и обновлять квоты могут только **Admin** и **Department Head** (для должностей своей ветки оргсхемы).

## 📡 API-Эндпоинты

### Словари метрик и назначения
- `GET /api/statistics/metrics` — получить список всех глобальных метрик.
- `POST /api/statistics/metrics` — создать метрику (только Admin).
- `DELETE /api/statistics/metrics?code=<CODE>` — удалить метрику (только Admin, если не используется).
- `GET /api/statistics/metric-to-post` — список назначений метрик должностям.
- `POST /api/statistics/metric-to-post` — назначить метрику должности (с указанием ответственного и дневной цели).
- `DELETE /api/statistics/metric-to-post?postId=<ID>&metricCode=<CODE>` — отменить назначение метрики.

### Ввод данных и Планирование
- `GET /api/statistics/daily-tracking?weekStart=YYYY-MM-DD` — получить сетку ежедневного учета для авторизованного сотрудника.
- `POST /api/statistics/daily-entry` — сохранить ежедневное значение метрики (пост, метрика, дата, значение).
- `GET /api/statistics/quotas` — получить список установленных планов (квот).
- `PUT /api/statistics/quotas` — установить или обновить квоту (цель) на период.
- `GET /api/statistics/records` — получить фактические периодические записи.
- `POST /api/statistics` — внести периодическое фактическое значение.

### Аналитика и Агрегация
- `GET /api/statistics/grid` — сводные данные за период (параметры `periodType`, `period`, `departmentId`, `responsibleUserId`, `myData`).
- `GET /api/statistics/series-30d?postId=<ID>&metricCode=<CODE>` — получить 30-дневный ряд данных и расчет WoW Growth.
- `GET /api/statistics/plan-vs-fact?metricCode=<CODE>&postId=<ID>` — план/факт за последние 7 дней для графиков.
- `GET /api/statistics/constructor?period=<PERIOD>` — представление для конструктора планов/фактов.
